Cognitive Architecture Optimization for the 'Nora' Learning Platform: Integrating Advanced Cognitive Science and Automated Calibration ModelsThe digital learning application 'Nora' represents a modern approach to learning technology, utilizing the Free Spaced Repetition Scheduler (FSRS-6) algorithm and Cepeda's ridgeline spacing model to schedule retention reviews. However, an analysis of the platform's current features reveals a critical gap between its backend scheduling and its frontend pedagogical methods.Nora operates with several limitations:Flashcard reviews rely on manual self-grading.Material is organized into rigid, domain-specific study patterns (blocked vocabulary and interleaved mathematics).Feedback on complex generative tasks is immediate and un-scaffolded.Gamification mechanics are restricted to basic pet-rearing and experience-point (XP) loops.There is no systematic support for micro-spacing, wakeful rest, or metacognitive calibration.This structural configuration creates several issues. First, manual card grading introduces metacognitive bias and high user friction. Users regularly struggle to distinguish between the qualitative rating states required by FSRS-6 (Again, Hard, Good, Easy), leading to inconsistent spacing intervals.Second, immediate correct-answer feedback during generative tasks serves as a cognitive crutch. This immediate feedback bypasses the cognitive effort required to identify and correct errors, leading to a false sense of mastery and shallow conceptual encoding.Third, the current gamification model risks triggering the overjustification effect. By replacing the internal motivation of learning with transactional external rewards, the platform encourages superficial completion behaviors rather than deep conceptual engagement.To address these limitations, this report outlines a comprehensive redesign of Nora's cognitive architecture. The proposed framework transitions the platform from a basic flashcard application into a highly integrated, psychologically optimized cognitive development environment.Practice Testing Formats and Retrieval DynamicsNora’s current reliance on standard flashcard retrieval represents a limited view of testing formats. A comparative analysis of practice testing formats highlights the distinct cognitive mechanisms triggered by free-recall flashcards, multiple-choice questions (MCQs), and short-answer production tasks.Practice testing consistently outperforms passive restudying across educational levels, a phenomenon known as the testing or retrieval-practice effect. Meta-analytic findings indicate a robust positive effect for testing, with Adesope et al. (2017) reporting a weighted Hedges’ $g = 0.61$ across 118 studies. Similarly, Rowland (2014) and Yang et al. (2021) identified a consistent medium-to-large positive effect ($g = 0.50$), with Yang et al. noting that the testing effect is particularly robust in applied classroom environments.                +--------------------------------------------------------+

The strength of the testing effect depends on the cognitive demands of the retrieval process. In university classroom studies, short-answer practice tests yielded superior performance on delayed criterial assessments compared to standard recognition tests.However, multiple-choice questions can be designed to match the learning benefits of short-answer formats. This is achieved by constructing competitive MCQs with highly plausible distractors that represent common user misconceptions.According to the discriminative-contrast hypothesis, choosing between competitive options forces the learner to actively compare similar concepts. Generative learning is highly effective, with Fiorella and Mayer (2015) reporting that prompting students to explain what they are studying improves post-test performance with a median effect size of $d = 0.61$ across 44 studies. Self-explanation encourages learners to generate inferences, connect new concepts with prior knowledge, and identify gaps in their understanding.However, prompting self-explanation without scaffolding can increase extraneous cognitive load, particularly for novice learners. This high load can lead to incomplete or incorrect explanations, which can reinforce misconceptions if left uncorrected.To address these limitations, Nora should replace its immediate feedback with a structured feedback timing model that supports generative learning. The platform can integrate several evidence-based alternatives to the Feynman technique.Scaffolded Self-Explanation Feedback AlternativesMenu-Based Self-Explanation: Originating from Aleven and Koedinger's geometry Cognitive Tutor, this format requires users to explain their problem-solving steps by selecting rationales from a structured dropdown menu rather than typing open-ended text. This approach reduces extraneous cognitive load for novice learners while guiding them to use precise, domain-specific language.Open-Ended Self-Explanation with LLM-Generated Feedback: This method uses large language models to analyze the semantic content of a user's open-ended explanation and provide real-time, adaptive feedback. This feedback does not simply provide the correct answer; instead, it provides targeted prompts that guide the user to revise and correct their own work. This interactive approach has been shown to improve explanation quality on transfer tasks, particularly for complex, non-explanation-provided problems.Self-Explanation-Based Automated Feedback (SEAF): This system provides real-time automated feedback using customized messaging, automated quality assessments, and peer-generated exemplars. This tiered feedback provides clear guidelines that encourage users to improve the depth and accuracy of their explanations.Educational Explainable AI Tools (EXAIT): This model uses dynamic playback features (such as pen-stroke and input time-series analysis) to prompt users to self-explain their step-by-step problem-solving process. The user annotates their own work at key points to explain why they chose a specific step. This scaffolding helps students generate high-quality, conceptual questions rather than basic factual recall items.Optimal Interleaving Schedules and Switching RatiosNora's current study architecture uses a rigid model: vocabulary is blocked, and mathematics is interleaved. While this approach recognizes that mathematics benefits from interleaved practice, it ignores the nuanced scheduling rules that govern optimal interleaving across different domains and learner profiles.Interleaving is highly effective for perceptual category learning and problem-solving, with meta-analytic reviews identifying an average effect size of Hedges' $g = 0.67$ for category learning. This benefit occurs because interleaving requires the brain to continuously retrieve and select different problem-solving strategies, which strengthens the mental processes needed to identify and solve novel problems.However, interleaving can be challenging for novice learners. For those with low domain-specific prior knowledge, interleaving can cause cognitive overload, resulting in poor initial performance and frustration.To address this, Nora should implement a fading interleaving schedule. Under this schedule, new concepts are initially introduced in blocked blocks to allow the user to build foundational schemas. As the user's proficiency increases, the system gradually increases the switch rate, transitioning the material into a highly interleaved configuration.                     +---------------------------------------+

The system can then prioritize highly miscalibrated items—such as those the user was highly confident in but failed to retrieve—for immediate successive relearning.Neurocognitive Consolidation: Micro-Spacing and Wakeful RestNora's current study flow encourages users to transition immediately from one intense cognitive task to another. This design ignores the role of post-learning cognitive activity in memory consolidation.According to the opportunistic consolidation account, minimizing post-encoding cognitive demands provides ideal conditions for the hippocampus to replay and stabilize newly encoded memories. Conversely, engaging in demanding cognitive or physical tasks immediately after learning introduces retroactive interference, which can disrupt and weaken these fragile memory traces.Recent cognitive literature (2020–2026) highlights wakeful rest (WR) as an effective countermeasure to retroactive interference. Wakeful rest is defined as a brief period (typically 5 to 10 minutes) of relaxed, low-stimulus wakefulness with closed eyes.This quiet window reduces external sensory inputs, allowing the brain to enter an offline state that supports consolidation. The temporal distinctiveness account suggests that minimizing immediate post-learning interference increases the isolation of target memories in the brain's psychological memory space, making them easier to retrieve later.                 +-------------------------------------------------------------+

In experimental evaluations, a 6-minute period of wakeful rest immediately following a learning session significantly improved recall performance on a 24-hour delayed test. This benefit is particularly strong for abstract, low-concreteness verbal materials, where resting yielded substantially higher retention compared to active distractor tasks like listening to music.While some ecological studies note that self-guided online users can struggle to maintain a restful state, incorporating guided, application-enforced wakeful rest directly into the study loop addresses this compliance bottleneck. Enforcing a brief, low-stimulus break after intense study sessions provides a highly scalable way to improve long-term retention.Self-Determination Theory (SDT) Gamification OverhaulNora's current gamification model relies on pet-rearing mechanics and experience points (XP). Implicit Spaced Repetition Grading LogicStandard FSRS-6 applications require the user to manually select an evaluation grade:Again (1): Total recall failure.Hard (2): Correct recall with significant effort.Good (3): Correct recall with moderate effort.Easy (4): Instant, fluent recall.To eliminate this manual step, the platform can calculate an implicit performance score ($S_{\text{perf}}$) using three real-time telemetry metrics:First-Keystroke Latency ($T_{\text{lat}}$): The time in milliseconds from question presentation to the first user input. This measures semantic retrieval speed, bypassing execution time.Correctness ($C_{\text{norm}}$): Normalized character-level Levenshtein distance between the user's input and the target answer:$$C_{\text{norm}} = 1 - \frac{\text{Levenshtein}(\text{Input}, \text{Target})}{\max(\text{len}(\text{Input}), \text{len}(\text{Target}))}$$Typing Dynamics ($K_{\text{back}}$): The count of backspace or delete keystrokes, which indicates uncertainty during retrieval.The system maps these metrics to FSRS-6 input states using the following rule system:$$\text{Implicit Grade} = \begin{cases} 
1 \ (\text{Again}) & \text{if } C_{\text{norm}} < 0.85 \\
2 \ (\text{Hard}) & \text{if } C_{\text{norm}} \ge 0.85 \ \text{and } (T_{\text{lat}} > 6000\text{ ms} \ \text{or } K_{\text{back}} > 3) \\
4 \ (\text{Easy}) & \text{if } C_{\text{norm}} \ge 0.95 \ \text{and } T_{\text{lat}} \le 2500\text{ ms} \ \text{and } K_{\text{back}} = 0 \\
3 \ (\text{Good}) & \text{otherwise}
\end{cases}$$This design automates FSRS-6 scheduling, removing grading friction while maintaining data integrity for the scheduling engine.Database Schema ArchitectureThe following PostgreSQL schema supports successive relearning sessions, student-authored questions, metacognitive calibration, and the implicit metrics log.SQL-- PostgreSQL Database Schema for upgraded Nora Platform

-- Core Cards Table updated with FSRS-6 parameters
CREATE TABLE cards (
    card_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL,
    deck_id UUID NOT NULL,
    cue TEXT NOT NULL,
    target TEXT NOT NULL,
    difficulty NUMERIC NOT NULL DEFAULT 5.0, -- 'D' in DSR Model
    stability NUMERIC NOT NULL DEFAULT 1.0,  -- 'S' in DSR Model
    retrievability NUMERIC NOT NULL DEFAULT 1.0, -- 'R' in DSR Model
    lapses INT NOT NULL DEFAULT 0,
    state INT NOT NULL DEFAULT 0, -- 0: New, 1: Learning, 2: Review, 3: Relearning
    due_date TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    last_review_date TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- Real-time telemetry log for automated FSRS grading
CREATE TABLE review_implicit_telemetry (
    review_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    card_id UUID REFERENCES cards(card_id) ON DELETE CASCADE,
    first_keystroke_ms INT NOT NULL,
    total_duration_ms INT NOT NULL,
    backspace_count INT NOT NULL DEFAULT 0,
    normalized_correctness NUMERIC(3,2) NOT NULL,
    derived_grade INT NOT NULL, -- 1: Again, 2: Hard, 3: Good, 4: Easy
    reviewed_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- Successive Relearning intra-session tracker
CREATE TABLE successive_relearning_sessions (
    session_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL,
    deck_id UUID NOT NULL,
    started_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMPTZ,
    mastery_criterion INT NOT NULL DEFAULT 1 -- required consecutive correct recalls
);

-- Tracker for active items within a successive relearning session (Dropout logic)
CREATE TABLE session_items_queue (
    session_id UUID REFERENCES successive_relearning_sessions(session_id) ON DELETE CASCADE,
    card_id UUID REFERENCES cards(card_id) ON DELETE CASCADE,
    consecutive_correct_recalls INT NOT NULL DEFAULT 0,
    is_dropped BOOLEAN NOT NULL DEFAULT FALSE,
    PRIMARY KEY (session_id, card_id)
);

-- Metacognitive Calibration database tracker
CREATE TABLE metacognitive_calibrations (
    calibration_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL,
    card_id UUID REFERENCES cards(card_id) ON DELETE CASCADE,
    predicted_recall_probability NUMERIC(3,2) NOT NULL, -- Judgments of Learning (0.00 to 1.00)
    actual_recall_success BOOLEAN NOT NULL,
    measured_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP
);

-- Student-Generated Questions (SGQ) Engine schema
CREATE TABLE student_questions (
    question_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    author_user_id UUID NOT NULL,
    deck_id UUID NOT NULL,
    constructed_question TEXT NOT NULL,
    correct_option TEXT NOT NULL,
    distractor_1 TEXT NOT NULL,
    distractor_2 TEXT NOT NULL,
    distractor_3 TEXT NOT NULL,
    scaffold_prompt_type VARCHAR(50), -- e.g., 'signal_words', 'question_stems'
    peer_review_rating_average NUMERIC(3,2) DEFAULT 0.0,
    created_at TIMESTAMPTZ NOT NULL DEFAULT CURRENT_TIMESTAMP
);
