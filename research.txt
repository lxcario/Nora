Official YouTube Data API v3: The official API exposes a structured schema for interacting with video properties, but accessing actual caption tracks is heavily restricted. The captions.list endpoint returns metadata for available tracks, while the captions.download endpoint downloads the raw file. However, these methods are blocked for general public videos. The official API requires OAuth 2.0 authentication and only permits caption downloads for videos owned by the authenticated channel or where the user has explicit content-partner editing permissions. Additionally, it cannot access auto-generated subtitles.The Unofficial Scraping Path (youtube-transcript-api): This Python-based library retrieves transcript tracks without requiring developer API keys or browser automation tools like Selenium. It functions by mimicking a client-side browser request, fetching the HTML of the target YouTube page, and parsing the nested JSON configuration objects to locate the timedtext API endpoint. It then requests the raw subtitle XML data and formats it into structured JSON segments containing text, start (in seconds), and duration.The Node.js and TypeScript Landscape (youtube-transcript-plus): For unified Next.js API route integration, youtube-transcript-plus provides a native, zero-dependency client written in TypeScript. This library performs a multi-step fetch sequence:videoFetch: Initiates a GET request to download the YouTube watch page.playerFetch: Executes a POST request to Google's internal InnerTube API client to obtain streaming metadata and caption track URLs.transcriptFetch: Downloads and parses the raw XML file, outputting structured JSON segments.Private InnerTube API Client (youtubei.js): A highly robust TypeScript client that interfaces directly with YouTube's private InnerTube API. By utilizing this client, a Next.js backend can instantiate a session via Innertube.create() and request a video's metadata through youtube.getInfo(videoId). Specifically, Section 3 bans accessing the service using automated means (such as robots, botnets, or scrapers) unless permitted by robots.txt, prior written authorization, or applicable law. Scraping transcripts using libraries that make unauthorized HTTP requests to YouTube's caption servers violates these terms.Copyright & Derivative Works: Video transcripts are the intellectual property of the content creator. Auto-generated captions are created by YouTube's internal automated speech recognition (ASR) engines, but copyright ownership of the underlying spoken text remains with the author. Storing complete transcripts, running them through an LLM, and generating public study materials may be considered a copyright violation if the resulting content is not sufficiently transformative to qualify as fair use.Mitigation Strategies: To establish ToS compliance, developers can use the standard YouTube Embed player to display videos, which is fully authorized by YouTube's terms. To isolate a specific segment, the backend converts formatted inputs (e.g., MM:SS or HH:MM:SS) into absolute seconds. The conversion equation is defined as:$$T_{\text{seconds}} = (\text{Hours} \times 3600) + (\text{Minutes} \times 60) + \text{Seconds}$$For a user-specified interval $[t_{\text{start}}, t_{\text{end}}]$, where $t_{\text{start}} = \text{262 seconds}$ (04:22) and $t_{\text{end}} = \text{390 seconds}$ (06:30), the slicing logic filters the transcript array:$$\text{SegmentOffset} \ge t_{\text{start}} \quad \land \quad \text{SegmentOffset} < t_{\text{end}}$$The selected segments are compiled into a unified context payload. To ensure the LLM can reference specific timestamps, each sentence is prefixed with its relative offset:JSON[
  { "offset": 262.5, "text": "The light-dependent reactions occur within the thylakoid membrane." },
  { "offset": 268.2, "text": "Chlorophyll pigments absorb photons, exciting electrons to a higher energy state." }

YouTube IFrame API Component IntegrationTo control playback and track the active video time, developers can construct a custom Next.js player component that initializes the standard YouTube Player and communicates via the window interface:TypeScript"use client";

import React, { useEffect, useRef, useState } from "react";

interface YouTubePlayerProps {
  videoId: string;
  onPlayerReady?: (player: any) => void;
  onTimeUpdate?: (currentTime: number) => void;
}

export const YouTubePlayer: React.FC<YouTubePlayerProps> = ({
  videoId,
  onPlayerReady,
  onTimeUpdate,
}) => {
  const containerRef = useRef<HTMLDivElement>(null);
  const playerRef = useRef<any>(null);
  const intervalRef = useRef<NodeJS.Timeout | null>(null);

  useEffect(() => {
    // Dynamic script injection to prevent SSR errors
    if (!window.YT) {
      const tag = document.createElement("script");
      tag.src = "https://www.youtube.com/iframe_api";
      const firstScriptTag = document.getElementsByTagName("script")[0];
      firstScriptTag.parentNode?.insertBefore(tag, firstScriptTag);
    }

    (window as any).onYouTubeIframeAPIReady = () => {
      initializePlayer();
    };

    if (window.YT && window.YT.Player) {
      initializePlayer();
    }

    return () => {
      if (intervalRef.current) clearInterval(intervalRef.current);
      if (playerRef.current) playerRef.current.destroy();
    };
  }, [videoId]);

  const initializePlayer = () => {
    playerRef.current = new window.YT.Player(containerRef.current, {
      videoId: videoId,
      playerVars: {
        controls: 1,
        enablejsapi: 1,
        rel: 0,
      },
      events: {
        onReady: (event: any) => {
          onPlayerReady?.(event.target);
          startTracker(event.target);
        },
      },
    });
  };

  const startTracker = (player: any) => {
    if (intervalRef.current) clearInterval(intervalRef.current);
    intervalRef.current = setInterval(() => {
      if (player && typeof player.getCurrentTime === "function") {
        const time = player.getCurrentTime();
        onTimeUpdate?.(time);
      }
    }, 500);
  };

  return (
    <div className="relative w-full aspect-video rounded-xl overflow-hidden shadow-md">
      <div ref={containerRef} className="w-full h-full" />
    </div>
  );
};
To navigate to a specific timestamp from the notes interface, developers can reference the player object and invoke seekTo:TypeScriptconst jumpToTimestamp = (seconds: number) => {

PostgreSQL Database Schema in SupabaseIn Supabase, developers can model the relationships between videos, user notes, and timestamp ranges. To do this efficiently, the system uses the native PostgreSQL numrange (numeric range) datatype to store the start and end offsets. Range types enforce valid periods at the type level and allow the database to handle range overlaps and containment queries natively.SQL-- Enable the btree_gist extension to support hybrid primary key exclusion constraints
CREATE EXTENSION IF NOT EXISTS btree_gist;

-- Videos Table
CREATE TABLE public.videos (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    youtube_id VARCHAR(11) UNIQUE NOT NULL,
    title TEXT NOT NULL,
    channel_title TEXT,
    duration_seconds INTEGER NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Video Notes Table utilizing Numeric Ranges
CREATE TABLE public.notes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    video_id UUID REFERENCES public.videos(id) ON DELETE CASCADE,
    time_segment NUMRANGE NOT NULL, -- Store [start_seconds, end_seconds)
    note_content TEXT NOT NULL,
    markdown_content JSONB, -- Rich text AST representation
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Constraint: Prevent overlapping notes for the same user on the same video interval
ALTER TABLE public.notes
ADD CONSTRAINT prevent_user_note_overlaps
EXCLUDE USING gist (
    user_id WITH =,
    video_id WITH =,
    time_segment WITH && -- && represents the overlap comparison operator
);

-- Index ranges for ultra-low latency temporal lookups
CREATE INDEX idx_notes_time_segment ON public.notes USING gist (time_segment);

This means that while unofficial scraping libraries work in local development, they are highly fragile and prone to breaking in production without rotating residential proxies.Alternative High-Throughput Transcription Services: To ensure platform stability, developers can implement a fallback transcription pipeline.
