# Nora's Voice

This is how Nora speaks. Every label, tooltip, empty state, and notification should feel like it came from the same thoughtful companion.

---

## Principles

Nora is warm. Never childish.

Nora encourages. Never pressures.

Nora is curious. Never lectures.

Nora is honest. Never pretends certainty.

Nora celebrates growth. Never perfection.

Nora uses "we" and "us." We're studying together.

## Things we keep technical

Some terminology is industry standard. Users who care about the science should recognize it. We keep:

- FSRS (for settings / advanced info)
- Spaced repetition (in descriptions, not labels)
- Confidence calibration (in analytics)
- Practice exam
- Interleaving (in tooltips)

The emotional language wraps around them, not replaces them.

---

## Three layers of language

Every concept in Nora can be expressed at three depths:

**Layer 1 — Technical**
> FSRS schedules reviews based on memory stability.

**Layer 2 — Human**
> It shows you things right before you'd forget them.

**Layer 3 — Emotional**
> Your garden grows because you remembered today.

Surface Layer 3 in the UI. Keep Layer 1 accessible in settings/docs. Layer 2 is for tooltips and descriptions.

## Tone by context

### Morning arrival
> "Good morning. Ready to learn something new?"

### After a mistake
> "That one's still growing."

### After mastery
> "We finally figured it out."

### After a Feynman explanation
> "I think I understand because of you."

### Returning after a break
> "Welcome back. I missed studying with you."

### All quests done
> "All done for today. You earned this."

### Empty states
> "Ready to plant your first memory?"

### Error messages
> "Something went wrong. Let's try again."
> (Never: "Error 500: Internal Server Error")

### Loading
> "Thinking..."
> (Never: "Loading...")

---

## Avoid

- "Congratulations!" (too performative)
- "Excellent work!" (too corporate)
- "You failed." (never)
- "Outstanding!" (too loud)
- "Great job!" (acceptable sparingly, but prefer subtler)
- Exclamation marks in excess
- Robotic phrasing ("The system has determined...")
- Passive voice about the user ("Your session was recorded")

---

## Prefer

- "Nice progress."
- "Let's revisit this together."
- "That's becoming clearer."
- "This one is still growing."
- "I'm here. Whenever you're ready."
- "We figured it out."
- Second person ("you") for actions, first-person plural ("we") for learning

---

## The test

Every screen should pass these questions:

1. Does this encourage instead of pressure?
2. Does this tell a story instead of just reporting data?
3. Does the companion feel present?
4. Is there unnecessary technical jargon?
5. Does the user leave this screen feeling more capable than when they arrived?

If a screen fails, keep iterating.

---

## One rule above all

If someone asks "What does Nora feel like?", the answer should be:

> **Someone quietly believes you can learn this.**

That's the feeling. Everything else follows from it.
